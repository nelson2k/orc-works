# Config — `tauri.conf.json`

Tauri reads a single config file at compile time. The default is `tauri.conf.json`, but the CLI accepts `.json`, `.json5` (with `config-json5` feature), and `Tauri.toml` (with `config-toml`). The canonical JSON Schema is generated by `tauri-schema-generator` and ships at `crates/tauri-schema-generator/schemas/config.schema.json` — example files reference it via `$schema`.

## Top-level shape

A real config from `examples/api/src-tauri/tauri.conf.json`:

```json
{
  "$schema": ".../config.schema.json",
  "productName": "Tauri API",
  "version": "1.0.0",
  "identifier": "com.tauri.api",
  "build":   { ... },
  "app":     { ... },
  "plugins": { ... },
  "bundle":  { ... }
}
```

The four block keys are the load-bearing ones:

| Key | Purpose |
| --- | --- |
| `build` | How the frontend is produced and where to find it. `frontendDist`, `devUrl`, `beforeDevCommand`, `beforeBuildCommand`, `removeUnusedCommands`. |
| `app` | Runtime behavior: `windows[]`, `security` (`csp`, `pattern`, `assetProtocol`, `freezePrototype`), `withGlobalTauri`, `macOSPrivateApi`, `trayIcon`. |
| `plugins` | Per-plugin config blobs, keyed by plugin name. Plugin authors define their own schemas. |
| `bundle` | Bundler config: `active`, `targets`, `icon[]`, `fileAssociations[]`, per-OS `windows`/`macOS`/`linux` blocks (WiX, NSIS, DMG, deb, etc.). |

## Security model (`app.security`)

- `csp` — string or object form. Tauri injects the right hashes/origins for IPC and asset protocols at build time.
- `pattern` — the **app pattern**. `"brownfield"` (default) is permissive; `"isolation"` runs a separate Isolation webview to mediate IPC and requires `options.dir` pointing at the isolation entry HTML.
- `assetProtocol` — opt-in `asset://` custom protocol, with scope `allow`/`deny` globs (supports `$APPDATA`, `$RESOURCE`, etc.).
- `freezePrototype` — JS prototype hardening.

## Allowlist (v1) vs ACL (v2)

v1 used `tauri.allowlist`. v2 replaces it with a permission / capability model. The CLI commands for it are `tauri permission` and `tauri capability`; capability files live alongside `tauri.conf.json` and reference per-plugin permission identifiers. See [commands_ipc.md](commands_ipc.md) for the runtime side.

## Where in code

- Parsing types: `crates/tauri-utils/src/config.rs` (the `Config` struct, all options).
- Compile-time embedding: `crates/tauri-codegen` (reads the JSON, generates the `Config`).
- Schema generator: `crates/tauri-schema-generator`. The shipped schema is checked into the repo under `crates/tauri-schema-generator/schemas/`.

## Bundle targets

`bundle.targets` accepts `"all"` or an array of: `app`, `dmg`, `appimage`, `deb`, `rpm`, `msi`, `nsis`, `updater`. The bundler ignores targets not applicable to the host OS — there is no cross-compile.
