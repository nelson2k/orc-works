# Repo Map

Source inspected at `repos-folder/PyMuPDF`.

## Top level

- `README.md` - product overview, quick starts, feature list, FAQ.
- `READMEb.md` - shipped as the README inside the `PyMuPDFb` wheel (MuPDF shared libs).
- `READMEd.md` - shipped as the README inside the `PyMuPDFd` wheel (MuPDF build-time files).
- `COPYING` - AGPL license text.
- `changes.txt` - changelog.
- `setup.py` - build logic; also generates `src/_build.py` at install time with version/git info.
- `pyproject.toml` - PEP 517 entry: `requires = ['pipcl']`, `build-backend = "setup"`, `backend-path = ["."]`.
- `pytest.ini` - pytest config (`python_files = tests/test_*.py`, `addopts = --tb=native`).
- `.readthedocs.yaml` - RTD build config (Ubuntu 20.04, Python 3.13, Sphinx).
- `valgrind.supp` - memory tooling suppressions.

## Source (`src/`)

Main implementation. `import pymupdf` resolves here.

- `__init__.py` (~25.9k lines) - the main API surface. Generated/maintained merge of SWIG output and hand-written code. Contains every public class: `Annot`, `Archive`, `Colorspace`, `DisplayList`, `Document`, `DocumentWriter`, `Font`, `Graftmap`, `Link`, `Matrix`, `IdentityMatrix`, `linkDest`, `Widget`, `Outline`, `Page`, `Pixmap`, `Point`, `Quad`, `Rect`, `Shape`, `Story`, `TextPage`, `TextWriter`, `IRect`, `TOOLS`, plus error types and internal device wrappers. Also contains module-level helpers like `get_tessdata`, `pymupdf.open` (alias for `Document`), and the multi-document concurrent `get_text`.
- `utils.py` (~1.2k lines) - higher-level helpers attached to core classes. OCR logic lives here in `get_textpage_ocr`. Also `get_text` (the unifying wrapper), search helpers, and many shape/text utilities.
- `table.py` (~2.7k lines) - table detection implementation (`find_tables`) and result classes.
- `__main__.py` - command-line entry helpers (`python -m pymupdf ...`).
- `pymupdf.py` - 2-line shim: `from . import *`.
- `fitz___init__.py`, `fitz_table.py`, `fitz_utils.py` - legacy `fitz` compatibility shims (re-export `pymupdf` symbols).
- `_apply_pages.py` - multiprocessing helper for concurrent page processing (used by the module-level `get_text`).
- `_wxcolors.py` - the named color dictionary (`ALICEBLUE`, `AQUAMARINE`, ...).
- `extra.i` - SWIG interface file for the extra C++ extension.
- `pipcl.py` - vendored build helper (PEP-517 backend logic).
- `wdev.py` - Windows Visual Studio toolchain discovery (used by the build).
- `_build.py` - NOT in the source tree; generated by `setup.py` at install time. Contains `pymupdf_version`, `pymupdf_version_tuple`, `mupdf_location`, git sha, swig version.

## Classic SWIG sources (`src_classic/`)

Older "classic" build path, kept for compatibility. SWIG `.i` files plus a parallel `__init__.py` and `utils.py`. Same public API surface; selected via build flags.

- `__init__.py`, `__main__.py`, `utils.py` - classic Python entry points.
- `fitz_old.i` (~15k lines) - main SWIG interface.
- `helper-*.i` - per-area SWIG helpers (annot, convert, devices, fields, fileobj, geo-c, geo-py, globals, other, pdfinfo, pixmap, portfolio, python, select, stext, xobject).
- `_config.h` - compile-time MuPDF config.
- `version.i` - version constants.

## Tests (`tests/`)

Pytest suite (~70 files). Notable for OCR/text work:

- `test_tesseract.py` - OCR / tessdata behavior (succeeds with `TESSDATA_PREFIX`, raises expected error otherwise).
- `test_textextract.py`, `test_textsearch.py`, `test_tables.py`, `test_textbox.py` - extraction/search/table/textbox coverage.
- `test_extractimage.py`, `test_rewrite_images.py`, `test_insertimage.py`, `test_imagebbox.py`, `test_imagemasks.py` - image handling.
- `test_threads.py` - thread behavior probe.
- `test_general.py`, `test_pixmap.py`, `test_annots.py`, `test_drawings.py`, `test_widgets.py`, `test_metadata.py`, `test_insertpdf.py`, `test_pagedelete.py`, `test_toc.py`, `test_crypting.py`, etc.
- Many `test_<issue#>.py` files - regression tests tied to upstream bug numbers.
- `conftest.py` - auto-installs test packages on first run (`pytest fontTools pymupdf-fonts flake8 pylint codespell mypy pipcl pillow psutil`), wraps every test with `wrap()` which checks `pymupdf.TOOLS.mupdf_warnings()` is empty and that tests don't mutate `set_small_glyph_heights`.
- `gentle_compare.py`, `run_compound.py`, `util.py` - shared test helpers.
- `resources/` - PDF/image fixtures (English, Chinese, scanned PDFs, OTF font, JPG, etc.).
- `README.md` - test setup notes.

## Build / release scripts (`scripts/`)

- `test.py` - developer build/test driver. Supports local MuPDF checkout, git URL/branch sources, pytest invocation.
- `gh_release.py` - CI wheel build via `cibuildwheel`. Defines the canonical `test_packages` string (used by `tests/conftest.py`). Produces the two-wheel split: `PyMuPDFb` (shared MuPDF libs, python-version-agnostic) + `PyMuPDF` (Python-specific bindings).
- `sysinstall.py` - Linux system-wide install path (build+install MuPDF and PyMuPDF into a root prefix, run tests with `LD_PRELOAD`/`PYTHONPATH`).
