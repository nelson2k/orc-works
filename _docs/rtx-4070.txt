DAILY WORKFLOW — RUNNING A LONG CONVERSION IN TMUX
--------------------------------------------------
Why tmux: if your SSH session drops, plain `python run.py` dies with it.
tmux keeps the process alive on the remote so you can detach + reattach.

Install (if missing):

  sudo apt install -y tmux

Start a named session and kick off a run:

  rtx-4070
  cd ~/dev/017-orc-works
  tmux new -s book
  source marker-code/venv/bin/activate
  python run.py --full-vram          # full book, 12 GB GPU defaults
  # or: python run.py --page-range 0-9 --full-vram   (quick smoke)
  # or: python run.py "pdfs/Other Book.pdf" --full-vram

Detach (process keeps running):
  Ctrl+B  then  D

Reattach later (from any fresh SSH):
  rtx-4070
  tmux attach -t book

Kill the session for real:
  exit                              # from inside
  tmux kill-session -t book         # from outside

List sessions:
  tmux ls

Useful inside tmux:
  Ctrl+B  then  [   -> scroll mode (arrows / PgUp/PgDn); q to exit
  Ctrl+B  then  D   -> detach (most-used)
  Ctrl+B  then  C   -> new window in the same session
  Ctrl+B  then  N   -> next window

GPU MONITORING
--------------
Open a second SSH session (no tmux needed for this), then:

  watch -n 2 nvidia-smi               # refreshes every 2s
  nvidia-smi --query-gpu=memory.used,memory.total --format=csv -l 2

While `run.py` is running you should see ~5-8 GB used (with --full-vram).
If you ever see >11.5 GB you're close to OOM and should drop batch sizes.


COMMON GOTCHAS
--------------
- Folder is "017-orc-works" on the remote (matches the GitHub repo's typo),
  but "017-ocr-works" on the local Windows box. Be consistent within each.
- `run.ps1` is Windows-only. On Linux just call `python run.py ...` directly
  after activating the venv.
- `pdfs/` is gitignored on local but was committed before that rule existed,
  so it came down with `git clone` and is also tracked on the remote. That's
  fine — saved an rsync.
- `marker-code/local.env` is gitignored (matched by `*.env`); never appears
  in commits. The remote's copy lives only in `~/dev/017-orc-works/marker-code/local.env`.
