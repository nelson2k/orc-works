# Style and theme

Three layers of styling:

1. **`StyleSetter`** ([StyleSetter.go](../../repos-folder/giu/StyleSetter.go)) — a builder that captures a stack of style + color + plot-style pushes, then applies them around a sub-tree.
2. **Direct `Push*` / `Pop*` helpers** ([Style.go](../../repos-folder/giu/Style.go)) — low-level wrappers over `imgui.PushStyleColor` / `imgui.PushStyleVar`.
3. **CSS stylesheets** ([CSS.go](../../repos-folder/giu/CSS.go)) — parsed at startup, applied automatically via tag names.

## Built-in themes

```go
giu.DefaultTheme()    // *StyleSetter
giu.LightTheme()      // *StyleSetter
```

`DefaultTheme` is the dark scheme used unless you override. `LightTheme` is its light counterpart. Apply via `wnd.SetStyle(giu.LightTheme())`.

## StyleSetter API

```go
ss := giu.Style().
    SetColor(giu.StyleColorWindowBg,  color.RGBA{R: 25, G: 25, B: 30, A: 255}).
    SetColor(giu.StyleColorButton,    color.RGBA{R: 80, G: 120, B: 220, A: 255}).
    SetStyle(giu.StyleVarFramePadding,  6, 4).
    SetStyleFloat(giu.StyleVarFrameRounding, 4).
    SetFont(myFont).
    SetFontSize(14).
    SetDisabled(false).
    To(
        giu.Button("Save"),
        giu.Button("Cancel"),
    )
```

Style enums are in `StyleIDs.go` and `stylecolorid_enumer.go` / `stylevarid_enumer.go` (generated by `enumer`):

- `StyleColorID`: Text, TextDisabled, WindowBg, ChildBg, PopupBg, Border, BorderShadow, FrameBg, FrameBgHovered, FrameBgActive, TitleBg, TitleBgActive, TitleBgCollapsed, MenuBarBg, ScrollbarBg, ScrollbarGrab, ScrollbarGrabHovered, ScrollbarGrabActive, CheckMark, SliderGrab, SliderGrabActive, Button, ButtonHovered, ButtonActive, Header, HeaderHovered, HeaderActive, Separator, SeparatorHovered, SeparatorActive, ResizeGrip, ResizeGripHovered, ResizeGripActive, Tab, TabHovered, TabActive, TabUnfocused, TabUnfocusedActive, PlotLines, PlotLinesHovered, PlotHistogram, PlotHistogramHovered, TableHeaderBg, TableBorderStrong, TableBorderLight, TableRowBg, TableRowBgAlt, TextSelectedBg, DragDropTarget, NavHighlight, NavWindowingHighlight, NavWindowingDimBg, ModalWindowDimBg
- `StyleVarID`: Alpha, DisabledAlpha, WindowPadding, WindowRounding, WindowBorderSize, WindowMinSize, WindowTitleAlign, ChildRounding, ChildBorderSize, PopupRounding, PopupBorderSize, FramePadding, FrameRounding, FrameBorderSize, ItemSpacing, ItemInnerSpacing, IndentSpacing, CellPadding, ScrollbarSize, ScrollbarRounding, GrabMinSize, GrabRounding, TabRounding, ButtonTextAlign, SelectableTextAlign

For implot:

- `StylePlotColorID`: line/scatter/bar colors
- `StylePlotVarID`: plot-specific spacing/sizes

Mutating methods:

- `SetColor(colorID, color.Color)` / `SetColorVec4(colorID, imgui.Vec4)` / `GetColor(colorID)`
- `SetStyle(varID, w, h)` — for `Vec2`-valued styles (`FramePadding`, `ItemSpacing`)
- `SetStyleFloat(varID, value)` — for scalar styles (`Alpha`, `FrameRounding`)
- `SetPlotColor(colorID, color)` / `SetPlotStyle(varID, w, h)` / `SetPlotStyleFloat(...)`
- `SetFont(*FontInfo)` / `SetFontSize(float32)`
- `SetDisabled(bool)` — graying out everything under

Composition:

- `(ss).Add(other *StyleSetter) *StyleSetter` — merges `other` on top
- `(ss).To(widgets ...Widget)` — apply to a Layout
- `(ss).Plots(widgets ...PlotWidget)` — apply to plot widgets specifically
- `(ss).Range(fn)` — apply across a range builder

Low-level lifecycle:

- `(ss).Push()` — push current state onto ImGui's stacks
- `(ss).Pop()` — pop matching pushes

The master window calls `Push` / `Pop` around the entire frame for its assigned theme.

## Direct Push helpers ([Style.go](../../repos-folder/giu/Style.go))

When you don't want a `StyleSetter`:

```go
giu.PushColorButton(color.RGBA{...})
giu.PushFramePadding(8, 6)
giu.PushItemSpacing(4, 4)
giu.PushItemWidth(120)
// ... draw stuff ...
giu.PopStyle()       // pop one style var
giu.PopStyleColor()  // pop one color
// or
giu.PopStyleV(3)
giu.PopStyleColorV(2)
```

Convenience setters (read current values):

```go
x, y := giu.GetWindowPadding()
w, h := giu.GetItemSpacing()
w, h = giu.GetItemInnerSpacing()
x, y = giu.GetFramePadding()
```

Cursor and text-wrapping:

```go
giu.PushTextWrapPos()
giu.PopTextWrapPos()
giu.AlignTextToFramePadding()
giu.SetMouseCursor(giu.MouseCursorTypeHand)
```

## Fonts

```go
fi := giu.Context.FontAtlas.AddFont("path/to/font.ttf", 16 /* px */)
// then:
giu.Style().SetFont(fi).SetFontSize(16).To(...)
// or
giu.PushFont(fi); /* ... */; giu.PopFont()
giu.PushFontSize(20)        // bump current font's size
```

`FontInfo` carries the parsed font; the atlas owns the texture. The atlas rebuilds incrementally whenever a frame mentions a glyph not in the atlas — see [FontAtlasProsessor.go](../../repos-folder/giu/FontAtlasProsessor.go).

## CSS stylesheets ([CSS.go](../../repos-folder/giu/CSS.go))

```go
sheet, err := giu.ParseCSSStyleSheet(`
  .main         { background-color: #161618; }
  .button       { background-color: #2266cc; color: white; padding: 8px 12px; border-radius: 4px; }
  .button:hover { background-color: #3377dd; }
`)
giu.Context.SetCSSStylesheet(sheet)
```

Each parsed CSS rule becomes a tag. Apply tags via widget builders:

```go
giu.Button("Save").Tag("button")
```

Tags compose with the underlying `StyleSetter`. The master window applies the `MainTag` stylesheet automatically each frame.

Supported CSS properties: ImGui-named ones (color, background-color, font-size, padding, margin, border-radius) plus a few extras. Colors are parsed by `mazznoer/csscolorparser`; the rest by `napsy/go-css`.

## Recipe: per-widget custom style

```go
giu.Style().
    SetColor(giu.StyleColorButton,        color.RGBA{R: 200, A: 255}).
    SetColor(giu.StyleColorButtonHovered, color.RGBA{R: 220, A: 255}).
    To(giu.Button("Delete").OnClick(deleteThing))
```

`Style().To(widgets...)` returns a widget that wraps the inner widgets with the matching push/pop pair.
