# Custom Models

Custom recognition model instructions are in `custom_model.md`.

To use a custom recognizer, three files must share the same base name:

```text
yourmodel.py
yourmodel.yaml
yourmodel.pth
```

Placement:

- `yourmodel.py` and `yourmodel.yaml` go in `user_network_directory`.
- `yourmodel.pth` goes in `model_storage_directory`.
- defaults are `~/.EasyOCR/user_network` and `~/.EasyOCR/model`.

Then initialize:

```python
reader = easyocr.Reader(["en"], recog_network="yourmodel")
```

The YAML provides:

- `imgH`
- compatible `lang_list`
- `character_list`
- `network_params`

The Python file must define the recognition model module importable by name. Internally, EasyOCR uses `importlib.import_module(recog_network)` for custom models.

The README recommends training custom recognition models with data generated by TextRecognitionDataGenerator or with the included `trainer/` scripts, which are adapted from `deep-text-recognition-benchmark`.
